System Policy Settings

There are three system policy tokens specific to the Causal Purge utility.

causal.PurgeLimitPerRun

In order to minimize duration, until experience has been gained that shows that more than the default 200 Causals can be purged in a single run, we have erred on the side of caution in establishing the default value. The value can raised or lowered at will.

causal.PurgeDeadlockRetryLimit

Given the parallel execution nature of the utility, we seek to minimize the possibility of having to stop the process due to excessive SQL Server DEADLOCK occurrences, measured against what could be a very low standard setting of 3 retries. The Purge Utility is set to retry up to 30 times. Lowering this count is deemed detrimental and unnecessary. It can be raised at will.

causal.PurgeRetentionDays

It is incumbent upon the implementation team to adjust this value at will. The data retained are the Purge table content, starting at Purge_Batch, and affecting each child table, and is for an entire batch, irrespective of the earliest start date-time of any Purge Step – retention starts from the date-time the Batch was last modified. Deletion of a Causal using the UI is at the discretion of the Customer.
