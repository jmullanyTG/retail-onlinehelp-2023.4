Verifying which plans are Default and which are Original Plan

A new Plan Sub Type Name column has been added as an available column to the Financial Plan Maintenance screen to supplement Plan Name, Plan Description, Plan Type Name, and Plan Status Name. The new column allows users to quickly and easily see which plans are Default and which are Original Plan.

Plan Sub Type Name can be added using the Choose Column option on the right-click menu.

Configuring the source of forecast level data

The source of forecast data to use (whether Inventory forecasts or aggregated level forecasts) is defined using new system policy Forecast Source (Module: Financial Planning). Level data should be chosen to expedite performance, but only if forecast jobs are set up to maintain hierarchy level data. The default is Inventory.

Rebalancing metrics based on configured measure sets

Previous Rebalancing functionality allowed for Sales to be rebalanced between plans. There were Receipt Retail, Receipt Units, and Receipt Cost checkboxes for Receipts to be rebalanced, but that functionality was never enabled. With this enhancement within v2023.4, users can now rebalance metrics based on configured measure sets.

Measure sets are configured in the Manage > Configuration > Merge Measure Sets screen.

Rebalancing should only occur between plans of the same plan type. The Rebalance Plan screen has been updated to include a Measure Set dropdown field. However, it is important to remember that the dropdown only displays measure sets relevant to the selected plan type. The Receipt Retail, Receipt Units, and Receipt Cost checkboxes are also now available for clicking.

Only the metrics selected for rebalance are affected within the destination plan. Any metrics unselected retain the values of the destination plan. As such, it is important to select all related metrics within a metric set for rebalance.

For example, when rebalancing Receipts, a user should check all of the Target Receipt metric checkboxes for the full set of data to rebalance. Selecting only Receipt Retail and Receipt Units without Receipt Cost applies the Receipt Cost of the destination plan to the newly copied Receipt Retail and Receipt Units values from the source plan, resulting in a newly calculated Receipt margin. Similarly, by not selecting Receipt Units, the Receipt AUR and Receipt AUC values will not be direct copies as the Receipt Units values in the destination plan become the divisor.

Rebalance functionality has not changed with the addition of this feature.

Notes:

  • If a measure set is picked along with any of the Sales / Sales Units / COGS / Receipt Retail / Receipt Units / Receipt Cost checkboxes, it is the measure set that is copied and not the Sales and Receipt specific metrics even if the Sales and Receipt checkboxes are selected.

  • Any values shown in the new Rebalance plan apply a SourceToTargetMultiplier congruent with ration of Sales / Total Sales within the mapping logic between plans.

  • Rebalance plans are only created with the time periods specified in the Rebalance Plan screen.

  • Within the Hybrid Retail Dual Markdown plan type, there are metrics with rolling calculations using the designated (checked) Season Begin period. If the source and destination plans do not have the same Season Begin period, there will be some drift in these rolling metrics, one of note being BOP Retail.
