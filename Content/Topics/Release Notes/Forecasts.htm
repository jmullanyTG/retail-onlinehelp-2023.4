Forecasts

Using AI/ML forecasted values to build MFP plans

MFP forecasting creates a forecast at the lowest MFP hierarchy level. Meaning, for an MFP plan that includes Mens > Mens Tops > Mens Tees > Mens Graphic Tees, with graphic tees being the lowest product level, then the forecast runs using the graphic tees grouping as the forecast source. This allows for a forecast to be generated based on product level history and projections and not tied to specific skus/locations. In addition to LY, LLY or a defined comparison time period, planners now has visibility to a sales projection as run through the forecast engine. The forecast results are the winning forecast method from the forecast tournament with LightGBM (AI/ML) included in that forecast tournament

Running forecasts for a plan or for all plans

A new type of forecast type, Merchandise Financial Plan Forecast Engine Task, has been developed that generates forecasts at a financial plan level and that does not go down to the inventory level. The forecasts can be generated for particular merchandise financial plans (MFP) or for all plans with users having the ability to specify which forecast parameters an MFP plan or all the MFP master plans should run under.

To generate forecasts for particular plans or to run a forecast for all plans, click Add to create a new task and select the appropriate option from the dropdown menu in the MFP Plan field on the Forecast Task dialog or click Edit next to the Engine Task field to update an existing task.

MFP forecast frequency

The MFP forecast frequency can be configured to run on a schedule such as weekly or monthly using two new job manager tasks. These tasks need to be configured so that TG JE's implementation of Gradient Boosting can analyze MFP plan level data.
