Wholesale Planning

Wholesale planners managing wholesale accounts can have accounts that split demand between multiple DCs. This functions similarly to TG JE's demand transference, though in this case wholesale accounts transfer their sales into DCs and not stores with demand aggregating to DCs only.

Wholesale accounts don’t have a sales history. Instead, these accounts use sell-ins (customer orders) in place of sales history for forecasting purposes and are managed by grouping multiple sell-ins together into major accounts. Only the major accounts are forecast and replenished.. For example,

  • Vandelay Industries (the primary wholesale account)

    • BrandA

    • BrandB

    • BrandC

    Vandelay Industries is the major account and is the account that is forecast and replenished.

Wholesale Planning

Using customer orders as sales history for Customers DMDEV-88190

A new weekly task, (WS) (Weekly) Generate Sales History from Customer Orders, uses customer orders that happened in the past as sales history if the customer order went to an account. The relationships defined in Customer Manager determine how Customer data rolls into which accounts.

Customer orders with dates in the future are ignored.

Site Clustering

There is a new Cluster Type in the Site Clustering Module labeled Wholesale Account Cluster. How sites are selected remains unchanged and is the same as with the other previously existing Cluster Types. Users do not select or create wholesale account clusters within the UI as these are automatically created within the import process. However, users will need to open new wholesale account cluster sets and click Calculate to assign the correct clusters. Snapshots continue as normal.

Editing a Cluster Set will show the Classifications created.

Site Manager

Reviewing the customers under an Account site type

Wholesale planners can, when managing wholesale accounts, see the customer names for sites of type Account by selecting a new widget Site Account Customer Relations. The customer names are read-only as changes to the Customer Account relationship are performed in Customer Manager.

Sites that are not of site type Account do not display any results.

Splitting Account demand into different DCs

A new widget labeled Site Account Source Site Contribution displays a cross product of items in the hierarchy and all DCs. This widget lets users review the default contribution factor (or split percentage) derived from past customer imports and override the default contributions on a particular hierarchy level as desired. However, an override itself cannot be overridden.

After a user enters a value in the Override field and clicks Save, the Clear Override field for that row is formatted orange. The Clear Override functionality clears overrides for all the DCs for the item. When overrides are cleared the default contribution percentage is used.

If 10,000 units have a 50/50 split between 2 DCs out of a total of 4 DCs, then 5,000 units are sent to each of the 2 DCs.

A new system policy Site Wholesale Account Distribution: ItemMaster Hierarchy ID defines the hierarchy used. (For more information, see the System Policies section.). Also, a new job task (WS) (Weekly) Calculate Source Site Percent Contribution for Accounts must be configured.

Supplier Manager

Reviewing construction type data

A new blade labeled Supplier Construction Type Capacity has been added to the main view in the Supplier Manager module. Selecting a supplier in the top grid refreshes the blade to display imported construction type data relating to that supplier.

A Construction type is an Item Master Group that defines what kind of manufacturing can product the item.

Each supplier can have multiple construction types and multiple construction groups associated with them. For example,

  • Supplier 1: (V-necks, T-shirts, and Tank Tops)

  • Supplier 2: (T-shirts, Dress-shirts, and Collar Shirts)

The supplier and the construction types that the supplier can provide are visible in a new blade within Order Manager labeled Supplier Capacity Report.

New system policy Supplier Constraint: Construction Type (Module: Order Manager) is used to configure the Attribute ID for the construction type.

Specifying capacities in buckets

Additionally, a second widget labeled Supplier Construction Type Capacity has been added to the Supplier Manager module. When a planner selects a supplier, time displays in the X-axis and construction type in the Y-axis.

 

Only the Total Supplier Quantity field is editable with all other columns being read-only. Planners can multi-select cells in the Total Supplier Quantity field to specify the capacity in that bucket.

 

New system policy Supplier Constraint: Bucket Type (Module: Order Manager) defines which Bucket ID to use that represents the time period. The bucket range is based on today and the value for existing system policy Future Days (Engine: Forecasts).

Merchandise Financial Planning

Within MFP, TG JE v2023.4 introduces a new Channel labeled Wholesale, with the ability to add sub-channel hierarchies beneath Wholesale for use in individual plans by account.

Including wholesale totals in the hierarchy

If the desired functionality is to include wholesale accounts in MFP but not plan them individually, then all that is necessary to do is include the Channel group in the MFP hierarchy configuration. Then Wholesale will be available when hierarchies are generated.

Including Individual wholesale accounts in the hierarchy

To include Wholesale in the Channel Hierarchy, the following steps must be taken:

  1. One of the Site Groups 1 - 20 should be used to represent each wholesale account. For example, Site Group 7 is given the friendly name Wholesale Account.

  2. Import Site Group values that exactly match each Account, and assign that SiteGroupID value to each wholesale account site.

    All Retail sites / DCs / other sites will have value of 0, No SiteGroup.

  3. Include the Site Group level in the Global Groups configuration in MFP.

Users can then view individual Wholesale Accounts in the MFP hierarchy.

Sites and customer accounts

Sites and Customer Accounts cannot co-mingle in MFP. Data for wholesale accounts can exist in their own hierarchy levels alongside retail hierarchy levels, but at the leaf level, they cannot intersect. For example, if the brand is ToolsGroup Tees, the Toolsgroup Tees-owned retail site Irvine and the Customer Account called Irvine Sport Shop that resells ToolsGroup Tees branded items cannot both exist in the Total Company > USA > Southwest Channel.

Order Manager

Planners can now see all the different suppliers for a selected order line based on the order line's construction type (construction types have a 1-to-1 relationship to items), the production capacity of each supplier over time, and how much of the production capacity is being used already by authorized orders.

When an order line is selected, the new Supplier Capacity Report blade populates with all the suppliers that have the order line’s construction type. Planners can review how much production capacity is left for each supplier of the construction type and how much of the capacity is taken up by pending stock. Supply planners can also manage scenarios where all suppliers are unable to fulfill the order so that exceptions are managed effectively.

Supplier production constraints

Often times a wholesaler’s constraint lies with supplier production. In this scenario and bearing in mind each supplier can produce X amount of construction types a month, a supply planner:

  • looks at all the unconstrained orders today

  • adjusts the order quantity based on constraints such as production capacity or orders for a particular vendor has been particularly low.

Planners can run a new macro Apply Supplier Production Limitation and the values in the User Order Quantity field in the top grid update according to the supplier production limit of that construction type. The Supplier Capacity Report displays recommended orders for today without any supplier constraint applied. Planners can review and apply supplier constraints to recommended orders as needed.

Both Replenishment and Assortment Planning orders can have supplier constraints applied to them.

Order lines that go beyond a supplier's production capacity split with the excess overflowing to another supplier of the same construction type. The resulting multiple order lines have the same order date, but different order quantities. Order quantities that exceed all available supplier capacity default to a supplier called Unfulfilled Supplier which shows the unfulfillable capacity.

The Order Date defines which recommended order lines of the same construction type are processed first with the earliest order date going first. The order in which suppliers are used to fulfill a construction type is dictated by the supplier construction priority provided during import.

Dates do not change. Therefore, policies such as schedules and lead time do not apply. However, policy Inventory Policy: Order Multiple and Min Order Quantity does apply.
